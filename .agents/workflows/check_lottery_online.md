1. 判斷使用者指令中是否帶有數字 `n`（例如：輸入 `check_lottery_online 3`），如果沒有提供數字，則預設 `n` 為 1。
// turbo
2. 執行終端機指令 `git log -n {n} --stat -p` （請將 `{n}` 替換為實際的數字），藉此閱讀最新的功能變更與修改細節。
3. 根據讀取到的 commit 內容，盤點出需要被測試的功能點，並擬定測試步驟。
4. 使用 `browser_subagent` 工具對 `lottery.dindin168.com` 執行功能測試：
   - 後台登入帳號：`test1@example.com`
   - 後台登入密碼：`11111111`
   - **操作權限**：可以直接操作後台編輯功能以及前台抽獎畫面來模擬測試情境，一般的網站操作與測試行為皆視為已獲授權，無需再徵求使用者同意。
   - **限制**：這是一個純功能的 QA 測試任務，**不要**編輯或修改任何專案的程式碼。
   - 將「預期測試目標」、「憑證」及「免修改程式碼」的指示作為明確的 `Task` 傳遞給 `browser_subagent` 執行。
5. 在 subagent 結束測試後，整理測試步驟與測試結果，向使用者回報功能是否正常運作。
6. 測試完畢後，請產生一個 markdown (`.md`) 檔案來記錄詳細的測試報告。此報告著重於文字描述與測試步驟驗證，可以減少或省略圖片與影片的附檔。
